# Spatial Methods for Identifying Undocumented Historical Earthquake DamageReproducible code accompanying the academic paper.  This pipeline processes earthquake damage report data (instrumental or historical sources) into spatial near tables, then runs three approaches of models: **Linear Regression**, **K-Nearest Neighbours (KNN)**, and **Kriging** to estimate seismic intensity at unreported sites.---## Requirements|Component|Version|Notes||-|-|-||**Python**|3.9 +|||**ArcGIS Pro**|2.9 +|Provides the `arcpy` module (Phase 1 only)||NumPy|≥ 1.22|||pandas|≥ 1.4|||scikit-learn|≥ 1.0|||SciPy|≥ 1.7|||PyKrige|≥ 1.7|Phase 2C (Kriging) only|> **Phase 1 (preprocessing)** requires a licensed ArcGIS Pro `arcpy` environment.  > **Phase 2 (modelling)** runs in any standard Python environment — no ArcGIS needed — as long as you supply computed near table.Install the non-arcpy dependencies:```bashpip install numpy pandas scikit-learn scipy pykrige```---## Project Structure```earthquake_project├── config.py                   # Column names are configured per earthquake in `config.py` via the `fields` mapping. Two built-in presets are provided:**USGS DYFI reports (instrumental)** — standard USGS "Did You Feel It?" file:```City, State/Region, Country, Zip Code, MMI, Responses, Distance, Latitude, Longitude```**Historical** — manually compiled macroseismic catalogues:```SITE_NAME, POINT_X, POINT_Y, Damage```For datasets with different column names, define a custom fields mapping in `config.py` (see the commented template).---## Quick Start### 1\. Full pipeline (Phase 1 + Phase 2)Requires ArcGIS Pro. Add your earthquake to `config.py` (or use an existing one), then:```bashpython main.py --earthquake Nippes_2021```### 2\. Phase 2 only (skip preprocessing)If you already have a near table (e.g. `Dead_Sea_1927_near_table.csv`), place it in `data/` and run:```bash# All models (Linear + KNN only — Kriging needs Phase 1 output)python main.py --earthquake Dead_Sea_1927 --skip_preprocessing# Single modelpython main.py --earthquake Dead_Sea_1927 --skip_preprocessing --model linearpython main.py --earthquake Dead_Sea_1927 --skip_preprocessing --model knn```> **Note on Kriging:** Kriging requires projected XY coordinates (columns: `X`, `Y`, `int`) which are generated by Phase 1 and saved to `results/`. It cannot run with `--skip_preprocessing` unless the `_Damage_locations.csv` exists in `results/` with the required columns (from a previous Phase 1 run or manually added). To run kriging standalone, use the CLI directly:```bashpython -m models.kriging --csv results/Dead_Sea_1927_Damage_locations.csv        --name Dead_Sea_1927_Damage_locations --anisotropy_angle 2```### 3\. Run individual models directlyEach model module has its own CLI:```bash# Linear Regressionpython -m models.linear_regression --near_table data/Dead_Sea_1927_near_table.csv --name Dead_Sea_1927# KNN (filtered + unfiltered)python -m models.knn --near_table data/Dead_Sea_1927_near_table.csv --name Dead_Sea_1927# Kriging (needs projected XY — see note above)python -m models.kriging --csv results/Dead_Sea_1927_Damage_locations.csv        --name Dead_Sea_1927 --anisotropy_angle 2```---## Adding a New Earthquake1. Add an entry to `EARTHQUAKE_PARAMS` in `config.py` with the epicenter coordinates, anisotropy angle (for universal kriging), and a `fields` mapping.2. Set the `fields` key to one of the built-in mappings, or define your own:   * `DYFI_FIELDS` — for instrumental DYFI data (columns: `City`, `MMI`, `Latitude`, `Longitude`)   * `HISTORICAL_FIELDS` — for historical data (columns: `SITE_NAME`, `Damage`, `POINT_X`, `POINT_Y`)   * **Custom** — copy the template in `config.py`, rename it, and set the column names to match your CSV3. Set the `filters` key to control the spatial distance ranges used by Linear Regression and KNN:   * `INSTRUMENTAL_FILTERS` — epicentral and neighbour distances from 100 to 500 km (100 km steps)   * `HISTORICAL_FILTERS` — epicentral and neighbour distances from 50 to 300 km (50 km steps)4. Place the input CSV in `data/`.5. Run the full pipeline: `python main.py --earthquake <your_key>`.---## Methodology Summary### Phase 1 — Preprocessing (`preprocessing.py`)1. Load input CSV → ArcGIS point feature class (WGS 84), using the field mappings defined in `config.py` to locate lon/lat columns.2. Create a persistent site-ID field (`MY_FID`) for later use.3. Create epicenter point feature class (WGS 84).4. `arcpy.analysis.Near`- distance \& azimuth from each site to the epicenter.5. `arcpy.analysis.GenerateNearTable`— sites distance, angle, and rank.6. Project to UTM and add XY coordinates (used by the Kriging model).7. Export damage location table; convert intensity to numeric (Roman → integer for DYFI data, pass-through for historical data).8. Merge epicenter distance/angle attributes into the near table; compute `intensity_diff` and `abs_int_diff`.### Phase 2A — Linear Regression (`models/linear_regression.py`)For each combination of spatial filters (azimuth window × epicentral distance × neighbour distance):* Bin sites distances and compute mean intensity difference per bin.* Fit a linear model (distance → intensity difference) on training bins.* Predict test sites intensity by applying the fitted slope to each neighbour's intensity, then averaging the K nearest predictions.> **Distance filter ranges** are configured per earthquake in `config.py`. Historical events use shorter ranges (50–300 km) due to sparser, geographically concentrated data, while instrumental events use wider ranges (100–500 km) to match the broader spatial distribution of reports.### Phase 2B — KNN (`models/knn.py`)* **Filtered Radius-based KNN**: within the spatial filter envelope (azimuth + epicentral distance), average the intensities of all training neighbours within a distance radius.* **Filtered Fixed-k KNN**: within the same envelope, average the intensities of the K nearest training neighbours.* **Unfiltered Radius-based KNN**: no spatial filtering; average all training neighbours within a distance radius.* **Unfiltered Fixed-k KNN**: no spatial filtering; average the K nearest training neighbours.### Phase 2C — Kriging (`models/kriging.py`)Operates on the processed damage locations table (columns: `X`, `Y`, `int`) rather than the near table. Grid search over variogram model, number of lags, n\_closest\_points, and drift terms for both Ordinary and Universal Kriging using PyKrige.---## Output FormatAll result CSVs contain at minimum: `earthquake`, `model`, `mse`, and error-rate columns (`error_+-0.5`, `error_+-1`). Additional columns vary by model (e.g. `angle_range`, `variogram_model`, `drift_term`).---## LicenseResearch code — see the accompanying paper for citation information.
